> Arquivo de contexto para projetos de lançamento pago no mercado digital brasileiro.
> Aplicável a qualquer cliente da agência. A seção 14 traz exemplo de aplicação ao projeto Clincog.

## 1. O QUE É UM LANÇAMENTO PAGO

Um lançamento pago é um **low ticket com ascensão programada**.

A lógica central:
- Vender um ingresso barato (R$37 a R$97) para um evento ao vivo
- Entregar valor real no evento
- Fazer o pit (oferta) do produto principal no final do evento
- Usar order bumps para aumentar o faturamento e o comparecimento

### Por que funciona melhor que lançamento gratuito

| Vantagem | Explicação |
|---|---|
| Menor risco financeiro | O tráfego vai sendo pago durante a captação de ingressos |
| Leads mais qualificadas | Quem paga preenche as duas lacunas: tempo E dinheiro |
| Maior comparecimento | Compromisso financeiro = pessoa leva mais a sério |
| Menos problema com volume | Grupos menores, mais gerenciáveis, menos golpes |
| Tráfego com mais escala | Com risco reduzido, dá para investir mais em tráfego |

### As duas lacunas que precisam ser preenchidas para uma venda acontecer
1. **Disponibilidade de tempo** — a pessoa precisa estar disposta a investir tempo para aprender
2. **Disponibilidade de dinheiro** — a pessoa precisa estar disposta a investir dinheiro para resolver o problema

Uma isca gratuita preenche só a lacuna 1. O ingresso pago preenche as duas.

## 2. ESTRUTURA COMPLETA DO LANÇAMENTO PAGO

```
TRÁFEGO PAGO (Meta Ads + base orgânica)
        ↓
LP DE INGRESSO (lotes progressivos)
Lote 0 → Lote 1 → Lote 2 → Lote N
        ↓
ORDER BUMP 1 — Gravação/Replay da aula
ORDER BUMP 2 — Material complementar
(opcional) ORDER BUMP 3 — Produto de ascensão
        ↓
GRUPO VIP (WhatsApp ou Telegram)
Lembretes + aquecimento + conteúdo preparatório
        ↓
EVENTO AO VIVO (1 dia ou 3 dias de aulas)
Entrega real + transformação + pit do produto principal
        ↓
ABERTURA DE CARRINHO
Produto principal com condição especial
        ↓
(opcional) DOWN SELL
Produto mais acessível para quem não comprou o principal
```

## 3. PLANEJAMENTO FINANCEIRO

### Lógica dos números
- Vender só ingresso **não paga o tráfego e não dá lucro sozinho**
- ~65% das vendas são ingressos, mas só ~34% do faturamento vem deles
- O faturamento real vem dos order bumps + produto principal
- Ticket médio do ingresso ≠ ticket médio por cliente (LTV)

### Exemplo de composição de faturamento
```
Ingressos vendidos: 750
Ticket médio ingresso: R$57
Faturamento ingressos: ~R$42.000

Order Bump 1 (gravação): 20% de conversão → R$X
Order Bump 2 (material): 15% de conversão → R$X
Produto principal: 14-20% de conversão → R$X
Down sell: 7-10% de conversão → R$X

Resultado total: 3x a 5x o faturamento dos ingressos
```

### Como calcular o ticket médio dos ingressos
Soma de (preço de cada lote × vendas de cada lote) ÷ total de ingressos vendidos

### Como calcular o pacing diário
(Total de ingressos desejados) ÷ (dias úteis de venda) = vendas necessárias por dia

## 4. LOTES DE INGRESSO

### Lógica dos lotes
- Lotes progressivos criam **urgência natural e escassez real**
- A virada de lote é o **maior pico de vendas** do período de captação
- Comunicar a virada em todos os canais: criativos, grupo VIP, e-mail, stories

### Estrutura de lotes recomendada
```
Lote 0 (early bird): R$37-39
Lote 1: R$42-45
Lote 2: R$47-49
Lote 3: R$54-57
Lote 4: R$64-67
Lote 5+: R$74-97
```

### Boas práticas de lote
- Criar checkout de cada lote com antecedência
- Automatizar virada de lote sempre que possível
- Barra de progresso na LP (aumenta conversão)
- Comunicar escassez em todos os pontos de contato

## 5. ORDER BUMPS

### Regra de ouro
Order bumps têm **sempre dois objetivos**: faturamento adicional + objetivo estratégico (comparecimento, ascensão, retenção).

### Order Bump 1 — Gravação/Replay
- **Preço:** R$27 a R$97
- **Conversão esperada:** 9% a 20%
- **Objetivo estratégico:** aumentar comparecimento e converter no produto principal depois
- **Importante:** replay converte mais pro produto principal do que lead sem replay

### Order Bump 2 — Material complementar
- **Preço:** R$27 a R$47
- **Objetivo estratégico:** mais pontos de contato, reter o lead na memória da marca
- **Exemplos:** checklist, template, guia prático, planilha

### Order Bump 3 — Cashback/Crédito
- **Preço:** R$97 a R$197
- **Objetivo estratégico:** criar incentivo financeiro para comprar o produto principal
- **Lógica:** o valor pago vira desconto no produto principal — a pessoa "perde" dinheiro se não comprar

### Benchmarks de conversão de order bump
- Conversão acima de 10%: muito bom
- Conversão de 18-32%: excelente (possível com copy de checkout otimizada)
- Quanto maior o ingresso, maior tende a ser a conversão do order bump

## 6. PRODUTO PRINCIPAL (PIT)

### O evento existe para converter no produto principal
A aula ao vivo deve:
1. Entregar valor real (o evento é um produto por si só)
2. Evidenciar um problema que o participante ainda não resolveu
3. Mostrar que a transformação é possível
4. Apresentar o produto principal como o próximo passo natural

### Estrutura do pit
1. Mostre a transformação que o evento entregou
2. Aponte o problema/lacuna que ainda existe
3. Apresente o produto principal como solução
4. Mostre o que está incluído (tabela de valor)
5. Condição especial exclusiva para quem está ao vivo
6. Urgência e escassez reais

### Benchmarks de conversão do produto principal
- Conversão de 8-14%: resultado abaixo do planejado, mas ainda positivo
- Conversão de 14-20%: resultado esperado
- Conversão acima de 20%: excelente, especialmente com high ticket

### Cashback como gatilho de conversão
- Quem comprou o Order Bump de cashback tem ainda mais incentivo para comprar
- Apresentar o cashback como "dinheiro que vai expirar" aumenta urgência

---

## 7. DOWN SELL

### Quando faz sentido ter down sell
- Quando o produto principal é high ticket (acima de R$2.000)
- Quando a base é grande o suficiente para justificar o trabalho
- Quando existe um produto de menor ticket que cumpre parte da promessa

### Quando o down sell pode não valer a pena
- Bases pequenas geram pouco faturamento com muito trabalho
- Produto de baixo ticket + coprodução + nota fiscal pode não sobrar margem

### Estrutura do down sell
- Oferecer versão mais acessível do produto principal
- Preço de 30% a 50% do produto principal
- Conversão esperada: 7-10% de quem não comprou o principal

## 8. ABERTURA DE CARRINHO (PÓS-AULA)

### Cronograma recomendado de 7-10 dias
```
Dia 1 (noite da aula): Abertura do carrinho — pico de vendas
Dia 2-3: Subir nível de consciência, quebrar objeções
Dia 4: Escassez do bônus especial — segundo pico
Dia 5-6: Novos argumentos, cópias de venda
Dia 7: Último dia — maior pico de vendas
```

### Ações durante o carrinho aberto
- Live aberta de dúvidas (aumenta audiência orgânica)
- Disparos de API oficial com mensagens de serviço
- Quebra de objeções via WhatsApp (monitorar perguntas frequentes)
- E-mail marketing para quem não abriu as mensagens anteriores
- Retargeting para quem visitou a página de vendas

### Bônus de escassez
- Criar bônus que seja valioso mas que não eclipsa o produto principal
- Se o bônus parecer mais valioso que o produto, quem fica de fora desiste de comprar
- Exemplos: livro físico, encontro extra, material exclusivo, acesso antecipado

## 9. TRÁFEGO E CRIATIVOS

### Estrutura de campanha recomendada
1. Começar com campanha Advantage+ (público aberto/frio) com todos os criativos
2. Testar criativos individualmente (isolados) para identificar campeões
3. Reunir criativos campeões numa nova campanha Advantage+
4. Escalar os que têm melhor ROAS medido por UTM (não confiar só no gerenciador)

### Teste A/B obrigatório
- Sempre testar variações de headline na LP
- Criativo campeão + headline alinhada = explosão de conversão
- A promessa específica e temporal sempre supera a promessa genérica
  - ❌ "Aprenda a investir"
  - ✅ "Como montar uma carteira de aposentadoria em até 15 anos"

### Anúncio Single Shot (remarketing avançado)
- Criado para público ultra-quente (viu 75%+ dos criativos ou visitou o site)
- O anúncio aparece **uma única vez** por pessoa
- Copy: *"Esse anúncio só vai aparecer uma vez para você. Nunca mais vai aparecer essa condição."*
- Usar nos últimos dias da captação
- Testar com 2-3 preços diferentes

### Tipos de criativos para lançamento pago
| Tipo | Posição no Funil | Objetivo |
|---|---|---|
| Pergunta/Provocação | Topo frio | Parar o scroll, gerar identificação |
| Storytelling/Virada | Topo frio | Empatia e prova de transformação |
| Contra-intuitivo | Topo frio | Quebrar crença limitante |
| Tabela de valor | Meio/Fundo | Mostrar o custo-benefício |
| Social Proof/Screenshot | Topo/Fundo | Prova social orgânica |
| Meme de identificação | Topo frio | Alcance orgânico, custo baixo |
| Expert + preço | Fundo/Remarketing | Sensação de oferta individual |
| Urgência/Virada de lote | Fundo | Empurrar quem está em dúvida |
| Single Shot | Remarketing final | Conversão de lead quente |

### Estrutura de roteiro de vídeo
1. **GANCHO** — Para o scroll em 2 segundos. Pergunta, provocação ou afirmação contra-intuitiva
2. **CORPO** — Aprofunda a dor, conecta ao produto
3. **CTA** — Ação clara, direta, sem rodeio
4. **NOTA DE PRODUÇÃO** — Tom, ritmo, dado a inserir, onde usar no funil

## 10. COMPARECIMENTO

### Por que comparecimento é crítico
- Sem pessoas ao vivo, não há conversão no produto principal
- Volume de pessoas ao vivo é proporcional ao resultado do pit
- Meta: fazer 1 milhão sem high ticket requer 2.000-3.000 pessoas ao vivo

### Estratégias para aumentar comparecimento
- Ingresso barato aumenta volume (mais pessoas)
- Grupo VIP com conteúdo de aquecimento aumenta comprometimento
- Lembretes sequenciados no grupo
- Replay com acesso limitado (quem não assiste ao vivo perde)
- Order bump de gravação cria incentivo para assistir (pagou, vai querer usar)

### Formato do evento
- **1 dia de imersão:** menos cansativo para o expert, mais difícil para o participante parar tudo
- **3 dias de aulas menores:** mais fácil de assistir, mais oportunidades de reforço
- **YouTube como plataforma:** sem problema técnico, funciona bem para audiência mais velha
- **Zoom:** mais interativo, mas mais propenso a problemas técnicos

## 11. PROMESSA DO EVENTO

### O evento é um produto por si só
As pessoas pagaram. A promessa precisa ser cumprida. A entrega precisa ser real.

### Estrutura de promessa que converte
- "Como [atingir resultado X] em [tempo Y]"
- "Como resolver [problema Z] aplicando [solução simples]"
- Palavras que funcionam: **montar, criar, estruturar, construir, fazer**
- Palavras que não funcionam: aprenda, entenda, saiba, descubra (passivo demais)

### O participante precisa sair com algo pronto
- Um plano prático de aplicação
- Um material gerado durante o evento
- Uma clareza que ele não tinha antes
- Um problema resolvido ou significativamente avançado

## 12. ESTEIRA DE PRODUTOS (contexto estratégico)

### Os 4 pilares de uma esteira completa
1. **Eu te ensino** — low e middle ticket, pessoa aprende e executa sozinha
2. **Eu automatizo** — software, planilha, ferramenta que resolve um processo
3. **Eu te acompanho** — mentoria, assessoria, acompanhamento personalizado
4. **Eu faço para você** — serviço completo, coprodução, execução

### Regra de ouro da esteira
O lançamento pago é sempre um produto de **"eu te ensino"** usado para vender o próximo nível da esteira (geralmente "eu te acompanho" ou "eu automatizo").

Se o próximo produto da esteira for também só "eu te ensino", a conversão cai — o participante sente que o evento já foi suficiente.

## 13. FERRAMENTAS E INFRAESTRUTURA

### Checklist de infraestrutura
- [ ] Checkout de cada lote criado com antecedência
- [ ] Order bumps configurados na plataforma (Hotmart, Kiwify etc.)
- [ ] Grupo VIP criado e automatizado
- [ ] LP com barra de progresso de lotes
- [ ] Traqueamento por UTM configurado (não confiar só no gerenciador)
- [ ] Dashboard de acompanhamento de vendas por dia
- [ ] Campanha de remarketing configurada (single shot)
- [ ] API oficial configurada para disparos

### Métricas para acompanhar diariamente
- Ingressos vendidos por dia vs. pacing planejado
- Custo por venda de ingresso (CPV)
- ROAS por criativo (medido por UTM)
- Conversão de order bumps
- Comparecimento ao vivo
- Conversão do produto principal

## 15. GLOSSÁRIO

| Termo | Significado |
|---|---|
| Low ticket | Produto de entrada, preço baixo (R$9 a R$197) |
| Pit | Momento da oferta do produto principal durante o evento |
| Order Bump | Produto adicional oferecido no checkout (antes da confirmação) |
| Down sell | Oferta de produto mais barato para quem não comprou o principal |
| Up sell | Oferta de produto mais caro após a compra |
| Pacing | Ritmo de vendas diário necessário para bater a meta |
| ROAS | Return on Ad Spend — retorno sobre o investimento em anúncios |
| CPV | Custo por venda |
| CPL | Custo por lead |
| LTV | Lifetime Value — valor total que um cliente gera |
| Single Shot | Anúncio de remarketing que aparece apenas uma vez por pessoa |
| Advantage+ | Formato de campanha Meta com público aberto e otimização automática |
| UTM | Parâmetro de rastreamento de URL para medir origem de conversões |
| API Oficial | API do WhatsApp Business para disparos em escala |
| Cashback OB | Order bump cujo valor vira desconto no produto principal |
